# Decision Log

## 2026-06-11 — PRD Creation

### Decisions Made

1. **项目定位**: 半公开运营工具，面向足球舆情行业代理/经销商体系。
2. **邀请码方案**: 明文输入（非 URL 参数），按隐性规则生成，包含生成者 ID 和目标角色类型。用户生成邀请码时需选择"发展代理"或"发展经销商"。
3. **角色确定**: 由邀请码自动解析，用户无需手动选择角色。
4. **资金流转模型**: 系统仅做"记账"计量，不处理真实资金流转。用户通过提现工单将记账余额转为实际收款。
5. **支付方式**: v1 全部线下 + 后台确认，不集成微信支付。
6. **提现方式**: 工单制，用户申请 → 管理员审核 → 线下打款 → 后台确认。
7. **角色升级**: v1 不支持经销商升级代理，需重新注册。
8. **长期奖励结算**: 默认按月，管理员可在后台配置（按月/按周/按天）。
9. **最低提现额度**: 默认 100 元，管理员可配置。
10. **邀请码格式**: 系统按隐性规则自动生成。
11. **数据看板**: 仅数字卡片 + 简单列表，不做图表。
12. **技术栈**: 微信小程序 + 独立部署 Web 管理后台 + 后端 API。
13. **账号销售验证**: 客户手机号需短信验证。
14. **管理员可配置项**: 所有佣金比例、金额参数均可在后台配置，修改仅对新业务生效。

## 2026-06-18 — PRD Major Revision (需求对齐)

### 变更背景

用户提供 `需求说明.md` 和 `用户旅途说明.md` 两份新文档，并发现已有"足球舆情分析系统"需要对接。经讨论后对 PRD 进行重大修订。

### Decisions Made

15. **邀请码简化**: 统一类型，不区分目标角色（原为发展代理/发展经销商/通用三种）。一次性使用，用后失效。只建立上下级关系，不影响角色。
16. **角色确定方式变更**: 由充值金额决定（888→888会员, 5000→经销商, 10000→代理），不再由邀请码决定。
17. **角色体系扩展**: 从 3 角色改为 4 角色（新增"888会员"）。
18. **充值独立制**: 各充值独立，不互斥，不升级。888会员可再充5000成为经销商（不退888）。
19. **两个场景定义**: 场景A（额度销售）不产生佣金；场景B（推荐充值）产生佣金。
20. **代理发展经销商后续收益**: 固定 133.2 元/笔，替代百分比长期奖励。
21. **长期奖励范围**: 仅限直接下级的全部收入，不穿透更深层级。
22. **普通用户推荐限制**: 仅 888 推荐有返佣（177.6元），推荐5000/10000不产生佣金。
23. **系统对接方案**: License Code 对接（不做SSO），手机号关联，两系统独立运行。
24. **License 机制**: 888会员开通后生成License，绑定手机号，一次性使用，舆情系统调用API验证。
25. **系统边界**: 销售系统负责注册→邀请码→角色→888开通→License生成；舆情系统负责License验证后的高级功能。

## 2026-06-21 — PRD v2 重大修订（业务系统对接）

### 变更背景

准备与舆情系统（业务系统）进行真实对接时，发现当前分销模型存在根本性矛盾：终端用户在业务系统开放注册，不在分销系统注册，导致"邀请码注册"场景不成立。经 Correct Course 流程讨论后对 PRD 进行 v2 重大修订。

### Decisions Made

26. **取消开放注册**：分销系统仅代理/经销商/管理员，不提供 `/register` 页面。`/login` 仅登录已注册用户，不创建新用户。
27. **邀请码 → 推荐码**：持久码，每人 1 个，可重复使用。用于支付时标识推荐人，不建立注册关系，只建立佣金归属关系。
28. **佣金 3 档 → 2 档**：去掉普通用户/888会员 20% 档。仅保留代理 55% / 经销商 40%。无推荐码 = 无佣金，平台收全款。
29. **License 不绑定邮箱**：生成时不绑定用户信息，激活时绑定业务系统用户标识。两系统用户体系完全独立。
30. **新增在线支付小闭环（FR-25）**：分销系统提供独立支付页面，支持 888/5000/10000 扫码支付。888→生成License；5000→创建经销商+额度+License；10000→创建代理+额度+License。
31. **超管创建种子用户**：管理员可直接创建代理/经销商，跳过支付流程，自动分配额度和推荐码。
32. **不做退款**：v1 不支持退款，支付成功后不可退。
33. **系统定位调整**：从"销售奖励系统"更名为"用户分销系统"，定位为与业务系统解耦的分销引擎。
